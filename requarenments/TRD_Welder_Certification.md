# **1\. Загальний опис системи**

Дана система призначена для автоматизації процесу атестації зварювальників відповідно до вимог НПАОП 0.00-1.16-96 («Правила атестації зварників»). Система забезпечує ведення реєстру груп атестації, введення даних атестації кожного зварника, автоматичну генерацію протоколів та посвідчень зварника встановленої форми.

Додавання нового підприємства виконується через існуючий в системі механізм управління підприємствами — без будь-якого дублювання інтерфейсу чи логіки.

# **4\. Бізнес-логіка та автоматичні розрахунки**

## **4.1. Генерація номера посвідчення**

Формат: {group\_number}.{order\_in\_group}-{attestation\_year}

Приклад: 3.15-26 означає: 15-й зварник у групі №3, рік атестації 2026 (взяти з дати групи).

Повний номер бланку посвідчення: ОД-1/{certificate\_number}

## **4.2. Генерація номерів протоколів контролю якості**

Кожен обраний метод контролю отримує номер виду: {group\_number}/{код\_методу}

| Метод контролю | Код | Приклад номеру протоколу (група №15) |
| ----- | ----- | ----- |
| Візуальний огляд | VT | 15/VT |
| Радіографічний контроль | RT | 15/RT |
| Ультразвуковий контроль | UT | 15/UT |
| Випробування на вигин | MT | 15/MT |
| Металографічні дослідження | MGT | 15/MGT |
| Додаткові методи контролю | IT | 15/IT |

Дата протоколів контролю якості вираховується з поля inspection\_date групи.

## **4.3. Строки дії посвідчення**

certificate\_valid\_until \= protocol\_date \+ 730 днів (2 роки)

next\_certification\_date \= protocol\_date \+ 730 днів (2 роки)

Базується на п.8.1 Правил атестації: посвідчення дійсне протягом двох років.

## **4.4. Вид атестації — відображення в протоколі**

| Значення ENUM | Відображення у протоколі (п.5) |
| ----- | ----- |
| primary | Первинна |
| additional | Додаткова |
| periodic | Періодична |
| extraordinary | Позачергова |

## **4.5. Логіка комбінованого зварювання**

Якщо is\_combined \= false: поля welding\_method\_2 і consumable\_2\_id залишаються NULL. У протоколі відображається один рядок для способу зварювання та один для електрода/дроту.

Якщо is\_combined \= true: обидва поля обов'язкові. Код атестації у посвідченні записується у вигляді 141\\111 Т BW W01 wm\\B t15 D219 PF ss nb\\mb, відповідно до прикладу 2 Правил атестації (п.7.7.5).

## **4.6. Обмеження на кількість деталей/діаметрів**

Якщо welded\_parts\_type \= 'plate': поля pipe\_diameter\_1/2/3 заблоковані (NULL). Доступні тільки thickness\_1/2/3.

Якщо welded\_parts\_type \= 'pipe': доступні всі шість полів. Користувач може заповнити від 1 до 3 товщин та від 1 до 3 діаметрів.

## **4.7. Область поширення атестації**

Поля admission\_\* розраховуються системою на підставі таблиць Правил атестації. 

## **4.8. Логіка коду атестації для посвідчення (п.7.7.1 Правил)**

Код атестації ({certification} у шаблоні) формується з 8 компонентів:

| Поз. | Компонент | Джерело даних |
| ----- | ----- | ----- |
| 1 | Спосіб зварювання | welding\_method\_1 \[\\welding\_method\_2\] |
| 2 | Тип деталі | welded\_parts\_type: «T» для труби, «П» для пластини |
| 3 | Тип шва | joint\_type: BW / FW |
| 4 | Група матеріалів | sample\_material.group\_code |
| 5 | Присадний матеріал | consumable\_1.coating\_type \[\\consumable\_2.coating\_type\] |
| 6 | Розміри зразків | thickness\_1..3, pipe\_diameter\_1..3 |
| 7 | Положення | welding\_position\_1 \[welding\_position\_2\] |
| 8 | Виконання шва | joint\_characteristics (ss nb, bs gg тощо) |

# **5\. Функціональні вимоги**

## **5.1. Управління групами (CRUD)**

* Створення групи: введення group\_number, protocol\_date, inspection\_date, certificate\_issue\_date, certificate\_issue\_location, вибір голови та до 5 членів комісії.

* Редагування групи: доступне до переходу статусу в 'completed'.

* Перегляд групи: список усіх атестацій у групі з можливістю відкрити кожну.

* Архівування групи: зміна статусу на 'archived'.

## **5.2. Управління атестаціями зварників (CRUD)**

* Додавання атестації: форма з усіма полями, описаними в розд. 3.2. Вибір групи є обов'язковим.

* Редагування атестації.

* Видалення атестації зі зменшенням order\_in\_group та перегенерацією certificate\_number.

* Перегляд атестації з можливістю генерації документів.

## **5.3. Генерація документів**

* Генерація протоколу засідання атестаційної комісії: один протокол — один зварник.

* Генерація посвідчення зварника: двостороннє, формат А5 (140×100 мм), у вигляді книжки.

* Завантаження документів у форматі .docx та/або .pdf.

* Генерація звіту по групі.

## **5.4. Управління довідниками**

* CRUD для: commission\_members, regulatory\_documents, sample\_materials, welding\_consumables.

* Soft-delete через поле is\_active (архівовані записи не з'являються у формах, але зберігаються в БД).

* Сортування переліків через sort\_order.

## **5.5. Управління шаблонами документів**

Реалізується на сторінці налаштувань системи. Шаблони згруповані за трьома типами:

* Протокол засідання атестаційної комісії (protocol)

* Посвідчення зварника (certificate)

* Протокол звітний (report\_protocol)

Для кожного типу доступні наступні дії:

* Перегляд переліку шаблонів: назва, дата завантаження, статус (активний / неактивний).

* Завантаження нового шаблону: користувач вказує назву та завантажує файл .docx. Новий шаблон за замовчуванням неактивний.

* Активація шаблону: користувач обирає один шаблон зі списку як активний. Система автоматично знімає позначку активного з попереднього шаблону того самого типу (swap в одній транзакції).

* Видалення шаблону: заборонено для активного шаблону. Спочатку треба активувати інший, потім видалити.

* Перейменування шаблону: редагування поля name без зміни файлу.

Генерація документів (п.5.3) завжди використовує шаблон з is\_active \= true відповідного типу. Якщо активний шаблон відсутній — система виводить попередження і блокує генерацію.

| pipe\_diameter\_1 | Обов'язкове якщо welded\_parts\_type \= 'pipe' | Вкажіть діаметр труби |
| thickness\_1 | Обов'язкове (хоча б одне) | Вкажіть товщину зразка |
| inspection\_date (у групі) | Має бути ≤ protocol\_date | Дата контролю не може бути пізніше дати протоколу |
| order\_in\_group члени комісії | Від 1 до 5 | До групи можна додати не більше 5 членів комісії |
| theory\_score | Обов'язкове | Вкажіть результат теоретичного іспиту |

# **10\. Зміни в навігації застосунку (бокове меню)**

В рамках даної розробки бокове меню застосунку модифікується: існуючі пункти та новий модуль «Атестація» організовуються у згорнуті підменю. Структура меню після змін:

| Пункт верхнього рівня | Підпункти | Примітка |
| ----- | ----- | ----- |
| Фінанси | Договори; Рахунки; Акти; Звіти | Існуючий функціонал, об'єднується у підменю |
| Атестація | Групи; Зварники; Налаштування | Новий модуль. Детальніше — нижче |

## **10.1. Підменю «Атестація»**

| Пункт меню | Маршрут (route) | Опис сторінки |
| ----- | ----- | ----- |
| Групи | /attestation/groups | Перелік груп атестації. Створення нової групи, перехід до картки групи зі списком зварників. |
| Зварники | /attestation/welders | Перелік усіх атестацій зварників по всіх групах. Додавання нової атестації, фільтрація, пошук. |
| Налаштування | /attestation/settings | Сторінка налаштувань модуля атестації. Містить управління шаблонами документів (п.2.6, п.5.5) та редагування довідників: члени комісії, нормативні документи, матеріали зразків, зварювальні матеріали. |

## **10.2. Сторінка «Налаштування» модуля атестації**

Сторінка /attestation/settings об'єднує весь конфігураційний функціонал модуля і складається з таких вкладок або секцій:

| Секція | Вміст |
| ----- | ----- |
| Шаблони документів | Управління шаблонами трьох типів: Протокол, Посвідчення, Протокол звітний. Для кожного типу — перелік завантажених шаблонів, кнопка активації, завантаження нового, видалення. (Детально — п.5.5) |
| Члени комісії | CRUD довідника commission\_members: додавання, редагування, архівування голів та членів комісії. |
| Нормативні документи | CRUD довідника regulatory\_documents: шифр, назва, текст допуску, порядок сортування. |
| Матеріали зразків | CRUD довідника sample\_materials: група (W01–W11), марка матеріалу. |
| Зварювальні матеріали | CRUD довідника welding\_consumables: марка, тип покриття. |

Важливо: пункт «Налаштування» у боковому меню відноситься виключно до модуля «Атестація» і не перетинається із загальними налаштуваннями застосунку (якщо такі існують).

# **9\. Нефункціональні вимоги**

* Всі довідники підтримують soft-delete (поле is\_active). Видалення пов'язаних записів — заборонено, лише архівування.

* Генерація документів (.docx) — на сервері, передача клієнту як файл для завантаження.

* Всі поля типу ENUM мають підтримувати локалізацію (UA).

* Система зберігає лог змін (created\_at, updated\_at) для всіх основних сутностей.
